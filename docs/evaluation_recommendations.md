# Evaluation Recommendations

> Status: framework only, pending real evidence (see `evidence_synthesis.md`).

## Intended structure

Practical suggestions for teams building their *own* evaluation
pipelines, informed by methodological lessons from the two sibling
benchmarks specifically (not general evaluation advice) — for example,
lessons about classifier validation, judge independence, or paired
statistical testing that came up concretely in either repo's
methodology. These are lessons about *how to measure*, distinct from
`policy_recommendations.md`'s lessons about *what to do* given a
measurement.
