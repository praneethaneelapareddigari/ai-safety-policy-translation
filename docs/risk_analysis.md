# Risk Analysis

> Status: framework only — populated once `evidence_synthesis.md` has
> real content to reason about.

## Structure this section will follow

For each risk identified in the evidence synthesis, this section will
document:

1. **Observed pattern** — the specific measured result it's based on
   (with a direct link to the evidence table row)
2. **Severity considerations** — what's actually at stake if this pattern
   holds in production (data exposure, harmful content generation, unsafe
   automation, etc.), scoped to what the benchmark actually tested
3. **Confidence** — sample size, whether the result has been replicated,
   whether it's a mock/pilot artifact or a validated finding
4. **What would change this assessment** — the specific follow-up
   experiment that would confirm, refute, or refine the risk

This structure exists to prevent risk claims from outrunning the
underlying evidence's actual confidence level.
